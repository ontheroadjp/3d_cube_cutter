# 構造主体アーキテクチャ検証計画

## 1. 目的
構造主体への移行後に、
切断結果の正確性・教育支援機能・数値安定性を検証する。

---

## 2. 検証カテゴリ

### 2.1 SnapPointID
- パース / 正規化が仕様どおりか
- 端点表記の取り扱いが一貫しているか
- 不正 ID を正しく弾けるか

### 2.2 GeometryResolver
- Vertex/Edge/Face の座標が一貫しているか
- サイズ変更に対して比率が維持されるか
- Face 基底 (basisU/basisV) の向きが安定しているか

### 2.3 Cutter
- 同一 SnapPointID で切断結果が再現されるか
- 交点数・輪郭線の順序が正しいか
- 反転切断が構造的に正しいか

### 2.4 NetManager
- 面所属判定が構造情報に基づいているか
- 2D 投影結果が面の向きと一致するか

### 2.5 教育支援
- マーカー色が SnapPointID の種別に対応しているか
- 解説文が構造情報に基づいて生成されるか

---

## 3. 代表テストケース

- **三角形切断**: `[V:0, V:1, V:2]`
- **辺中点切断**: `[E:01@1/2, E:12@1/2, E:23@1/2]`
- **混合切断**: `[V:0, E:12@1/4, V:3]`

---

## 4. 回帰確認
- 既存バージョンとの切断面形状一致
- 既存プリセットの動作確認

---

## 5. 合格基準
- 切断面の頂点数と順序が想定と一致
- 展開図の切断線が対応する面に描画される
- SnapPointID を変えない限り結果が再現される

---

## 6. まとめ
- 構造主体移行後は ID と結果の一貫性が最重要
- 教育支援機能も検証対象に含める
