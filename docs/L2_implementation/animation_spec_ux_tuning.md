# AnimationSpec UX Tuning Guidelines

本ドキュメントは AnimationSpec を用いた展開アニメの UX 調整方針をまとめる。
学習用途であることを最優先とし、理解を妨げる演出を避ける。

---

## 1. 現行の分かりにくさ / ノイズ要因

- カメラが急激に切り替わると、面の移動方向が把握しづらい
- 複数面が同時に動くと視線が散り、切断面の理解が遅れる
- 展開後の全体像が画面外に出ると、面の位置関係が崩れる

---

## 2. AnimationSpec パラメータと UX 影響

### 2.1 duration
- 1面あたりの回転時間。短すぎると「どの面が動いたか」見失いやすい
- 推奨レンジ: 0.6s - 0.9s（学習用途の視認性優先）

### 2.2 stagger
- 面の開く間隔。0に近づくほど同時進行に見える
- 推奨レンジ: 0.5s - 0.8s（1面ずつ認知させる）

### 2.3 ease
- `easeOutCubic` は終端をゆっくり止められるため、面の位置確定が理解しやすい
- `linear` は教材用には単調で動きが硬く見えることがある

### 2.4 moveCamera
- 展開時は俯瞰（Top view）寄りに寄せると面の配置が見やすい
- 回転後の位置が安定するまでカメラ移動を重ねない

### 2.5 highlight / visibility
- 切断面や新しく展開された面を一時的に強調すると理解が早い
- 強調色は既存 UI と競合しない彩度に抑える

---

## 3. 調整方針（AnimationSpec への反映）

- `rotateFace` の `duration` と `stagger` を優先的に調整し、1面ずつの進行感を担保する
- `moveCamera` は展開開始直後に合わせて短めに入れる（長い追従は避ける）
- `highlight` は「面が開き終わった後」に短時間入れるのが効果的
- 学習モードでは text overlay を同期させる場合、面の停止タイミングに合わせる

---

## 4. チェックリスト

- 1面ずつ開いていることが視覚的に分かるか
- カメラ移動が面の回転を邪魔していないか
- 展開後の全体が画面内に収まっているか
- 強調表示が学習対象を指し示しているか
