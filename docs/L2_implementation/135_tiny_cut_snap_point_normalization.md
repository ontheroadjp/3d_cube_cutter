# #135 Tiny Cut SnapPointID Normalization

## 実装(技術)について
- cut 面の頂点順序は keepPositiveSide に依存するため、cut 面の法線を「保持する半空間の反対向き」に合わせて整列した。
- `Cutter.getResultFacePolygons` と `cutter/CutComputation` の両方で同じ向きルールになるよう調整した。

## 仕様(L1ドキュメント)について
- L1 の「断面 Face は通常 Face と同様に CCW 順序を持つ」方針に沿い、保持側の外向きが一貫するようにした。

## 数学的観点から
- 切断平面の法線方向は「正側」を定義するが、保持側の外向き法線はその逆向きになることがある。
- tiny cut の場合でも面の向き整合性を維持するため、保持半空間に応じて cut 面の法線を反転した。

## ユーザー体験(ユーザー価値)について
- 微小切断でも展開・検証が破綻せず、学習対象の一貫性が維持される。
