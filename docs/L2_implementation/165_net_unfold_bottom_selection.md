# 165 Net Unfold Bottom Selection

## 実装(技術)について
- 底面選択は既存の raycaster を流用し、faceMeshes を直接ヒットテストすることで構造主導の選択を維持した
- NetPlan 生成時に rootFaceId を渡し、BFS 展開順の起点を固定した
- カメラ位置は rootFace の normal/basisU/basisV から派生するため、座標主導にならない

## 仕様(L1ドキュメント)について
- Net 仕様と UI 仕様に同一の展開フロー（1〜6ステップ）を明記し、工程の再現性を確保した
- 90度固定の回転は立方体限定であり、一般多面体では dihedral 角を採用する前提を追記した

## 数学的観点から
- 展開角は dihedral angle に基づくため、cut face を含む多面体でも 180° に収束する
- rootFace を基準に法線/接線を算出し、基準面への整列と展開方向を統一した

## ユーザー体験(ユーザー価値)について
- 展開開始前にズームを調整し、展開中も画面内に収まる余白を確保した
- 展開開始前の斜め視点を固定し、展開途中のヒンジ関係が見える状態を確保した
- 展開完了後も斜め視点を維持し、展開の動きと形の両方を連続的に理解できるようにした
