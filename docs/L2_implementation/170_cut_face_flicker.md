# #170 Cut Face Flicker

## 実装(技術)について
- SSOT パイプラインでは Cube が面を描画しているため、CSG の resultMesh を同時に描画すると面が重複し z-fighting が発生する。
- Cut 実行時、SSOT の場合は resultMesh を非表示にし、構造主導の面描画に一本化した。

## 仕様(L1ドキュメント)について
- 構造主導の表示（SSOT → Derived → View）を維持し、座標由来のメッシュを正としない方針に沿う。

## 数学的観点から
- 同一平面上の面の重複描画は深度バッファの競合を招き、フリッカが発生する。
- 片方を非表示にして幾何学的な重複を解消した。

## ユーザー体験(ユーザー価値)について
- 切断後の面表示が安定し、学習中の視認性が向上する。
