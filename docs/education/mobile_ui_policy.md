# モバイルUI簡略化方針

## 1. 目的
タブレット/スマホで快適に操作できるよう、
UIや機能を条件付きで簡略化する。

---

## 2. 基本方針
- 画面幅が一定以下の場合にモバイルモードを有効化
- 重要機能だけを残し、補助機能は折りたたむ

---

## 3. 簡略化対象

- プリセット一覧をカテゴリ単位で折りたたみ
- 設定項目の一部を非表示（例: 面ラベル/辺ラベルの詳細設定）
- 展開図の表示はモーダルに集約
- ユーザープリセット管理は有料版のみ、かつ簡易一覧に縮小

---

## 4. 判定基準
- `window.innerWidth <= 768` をモバイル条件とする
- ただしタブレット横持ちはフル機能へ戻す余地を残す

---

## 5. まとめ
- モバイル最適化を優先し、必要最低限のUIを提供
- 保存機能などの重い操作は段階的に表示
