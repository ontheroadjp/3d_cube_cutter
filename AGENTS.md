## 作業を進め方（重要）

本プロジェクトでは堅牢なシステムが必要です。常に動く状態を保ちながら丁寧に作業を進め常に全体のドキュメントの整合性を保ちながら
ドキュメントを作成・更新し、テストケースで品質を担保する。

## git commit について
コミットメッセージは [Conventional Commits](https://www.conventionalcommits.org/) の規約に従い全て英語で表記します

## 共通ワークフロー（プロジェクト横断）
